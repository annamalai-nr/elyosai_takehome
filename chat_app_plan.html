01What we're building

A single-file async CLI chat application that:

  1. Reads model name from config.yaml, validates against allowed_models.csv, loads secrets from .env
  2. Accepts text input in a You: prompt loop
  3. Streams the LLM response token-by-token to the terminal
  4. Supports two tools via OpenAI function calling: get_weather and research_topic
  5. Shows pending state during tool calls ("Researching solar energy... (Ctrl+C to cancel)")
  6. Handles Ctrl+C: cancels the current turn task via asyncio.CancelledError, preserves partial output, returns to prompt
  7. Handles every API quirk discovered in the probes
  8. Maintains conversation history across turns
  9. Supports --validate mode: feeds fixture dicts through parsers to verify correctness without API calls

03Config & secrets loading

Per north_stars.md rule 2: "Model names are config-driven. Never hardcode model strings in calling code."

Startup sequence

  1. Load .env via python-dotenv.
  2. Read config.yaml → extract llm.model_name, elyos_api.base_url, elyos_api.api_key_env.
  3. Validate llm.model_name exists in allowed_models.csv with type=text. Fail loudly if not.
  4. Check os.environ[ELYOS_API_KEY] and os.environ[OPENAI_API_KEY] are set. Fail loudly if missing. Never print values.
  5. Init openai.AsyncOpenAI() and httpx.AsyncClient().

def load_config():

06Tool-call protocol (OpenAI Chat Completions)

This is the exact message sequence. Getting it wrong means tool calling silently breaks.

6.1 Streaming with tool calls

stream = await oai.chat.completions.create(
    model=model_name,     # from config.yaml
    messages=messages,
    tools=TOOLS,
    stream=True,
)

content_parts = []
tool_calls = {}       # index -> {id, name, arguments_buffer}

async for chunk in stream:
    delta = chunk.choices[0].delta
    if delta.content:
        print(delta.content, end="", flush=True)
        content_parts.append(delta.content)
    if delta.tool_calls:
        for tc_delta in delta.tool_calls:
            idx = tc_delta.index
            if idx not in tool_calls:
                tool_calls[idx] = {"id": tc_delta.id, "name": tc_delta.function.name, "args": ""}
            if tc_delta.function.arguments:
                tool_calls[idx]["args"] += tc_delta.function.arguments

6.2 After stream ends: execute tools and recurse

if tool_calls:
    # Step 1: append the assistant message WITH tool_calls
    messages.append({
        "role": "assistant",
        "content": "".join(content_parts) or None,
        "tool_calls": [
            {"id": tc["id"], "type": "function",
             "function": {"name": tc["name"], "arguments": tc["args"]}}
            for tc in tool_calls.values()
        ],
    })

    # Step 2: execute each tool, append role:"tool" messages
    for tc in tool_calls.values():
        args = json.loads(tc["args"])
        result_str = await execute_tool(client, cfg, tc["name"], args)
        messages.append({
            "role": "tool",
            "tool_call_id": tc["id"],
            "content": result_str,
        })

    # Step 3: recurse (make another streaming call with tool results)
    round_count += 1
    if round_count >= MAX_TOOL_ROUNDS:
        print("[Max tool rounds reached]")
        return
    return await stream_turn(oai, client, cfg, messages, round_count)
else:
    # No tool calls — final assistant text
    full_content = "".join(content_parts)
    print()
    messages.append({"role": "assistant", "content": full_content})
    return full_content

MAX_TOOL_ROUNDS = 5 prevents infinite recursion if the LLM keeps requesting tools. In normal use we expect 1–2 rounds (tool call + final response).

07Cancellation design revised

v1 was wrong. In asyncio.run(), SIGINT cancels the main task with asyncio.CancelledError, not KeyboardInterrupt. Relying on KeyboardInterrupt is brittle. v2 uses a custom SIGINT handler that cancels only the current turn task.

7.1 Mechanism

  1. main() installs a SIGINT handler via loop.add_signal_handler(signal.SIGINT, ...). This overrides asyncio.run's default SIGINT-cancels-main behaviour.
  2. Each turn runs as asyncio.create_task(stream_turn(...)).
  3. SIGINT handler calls turn_task.cancel() if a task is running. If no task is running (e.g. during input()), it raises KeyboardInterrupt to exit the app.
  4. main() catches asyncio.CancelledError from the turn task.

7.2 Partial output preservation

Per north_stars.md rule 5: "preserve partial LLM response in history."

  • stream_turn() accumulates content_parts as tokens arrive.
  • On CancelledError, main() checks if any content was printed.
  • If yes: keep the user message, append a partial assistant message marked " [interrupted]".
  • If no (cancelled during tool call before any text): remove the user message entirely.

7.3 Code sketch

_turn_task = None

def _on_sigint():
    if _turn_task and not _turn_task.done():
        _turn_task.cancel()
    else:
        raise KeyboardInterrupt

async def main():
    cfg = load_config()
    loop = asyncio.get_running_loop()
    loop.add_signal_handler(signal.SIGINT, _on_sigint)
    messages = [{"role": "system", "content": SYSTEM_PROMPT}]

    async with httpx.AsyncClient() as http_client:
        oai = openai.AsyncOpenAI()
        while True:
            try:
                user_input = await asyncio.to_thread(input, "\nYou: ")
            except (EOFError, KeyboardInterrupt):
                break
            if user_input.strip().lower() in ("quit", "exit", "q"):
                break

            messages.append({"role": "user", "content": user_input})
            print("Assistant: ", end="", flush=True)

            global _turn_task
            _turn_task = asyncio.create_task(
                stream_turn(oai, http_client, cfg, messages)
            )
            try:
                partial = await _turn_task
            except asyncio.CancelledError:
                print("\nCancelled.")
                if partial_content_so_far:
                    messages.append({"role": "assistant",
                                     "content": partial_content_so_far + " [interrupted]"})
                else:
                    messages.pop()
            finally:
                _turn_task = None

Note on partial_content_so_far: stream_turn needs a way to expose its accumulated content to the caller even on cancellation. A simple approach: store it on a mutable object (list) passed in, or use a module-level variable. The exact mechanism is an implementation detail.

08Data flow

Happy path (weather query)

User: "What's the weather in Tokyo?"
  |
  v  main() creates turn_task = stream_turn(messages)
  |
  v  OpenAI streams: tool_call(get_weather, {location: "Tokyo"})
  |  (tool_calls dict accumulates id + args from deltas)
  |
  v  Stream ends. Append assistant msg with tool_calls to messages.
  |
  v  execute_tool("get_weather", {location: "Tokyo"})
  |    -> prints "Looking up weather for Tokyo..."
  |    -> call_api(http_client, "/weather", {location: "Tokyo"})
  |    -> normalise_weather(response, "Tokyo")
  |    -> JSON envelope: {"source":"elyos_api","untrusted":true,"data":{...}}
  |    -> append {"role":"tool","tool_call_id":"...","content":"..."} to messages
  |
  v  Recurse: stream_turn(messages) again
  |
  v  OpenAI streams: "The weather in Tokyo is 28C and humid..."
  |    -> printed token-by-token
  |
  v  Append assistant message. Return to prompt.

Cancellation path

User: "Research quantum computing"
  |
  v  turn_task = stream_turn(messages)
  |  OpenAI streams tool_call -> execute_tool()
  |  prints "Researching quantum computing... (Ctrl+C to cancel)"
  |  call_api() -> awaiting httpx for 3-8s...
  |
  [User presses Ctrl+C]
  |
  v  SIGINT handler fires -> turn_task.cancel()
  v  CancelledError propagates through httpx -> stream_turn
  v  main() catches CancelledError
  v  No partial content was printed -> remove user message
  v  prints "Cancelled."
  v  Back to "You: " prompt. httpx client still healthy.

Throttle path

call_api() -> 200 OK, body: {"status":"throttled","retry_after_seconds":7}
  |
  v  prints "Rate limited, waiting 8s..."
  v  await asyncio.sleep(8)  (cancellable if Ctrl+C)
  v  retry same request
  v  200 OK with real data -> return parsed JSON

09Validator mode (--validate)

Per north_stars.md: "No pytest. Use simple Python test scripts." A --validate flag feeds fixture dicts through the parsers.

def validate():
    # Weather Shape A
    a = normalise_weather(
        {"location":"London","temperature_c":14.0,"condition":"Partly cloudy","humidity":67},
        "London")
    assert a["returned_location"] == "London"
    assert len(a["observations"]) == 1

    # Weather Shape B
    b = normalise_weather(
        {"location":"Paris","conditions":[
            {"temperature_c":13.2,"condition":"Overcast","humidity":88},
            {"temperature_c":12.2,"condition":"light rain","humidity":100}],
         "note":"Multiple conditions reported"},
        "Paris")
    assert len(b["observations"]) == 2

    # Weather fuzzy match
    m = normalise_weather(
        {"location":"Marseille","temperature_c":20.0,"condition":"Sunny","humidity":55},
        "Mars")
    assert m["requested_location"] == "Mars"
    assert m["returned_location"] == "Marseille"

    # Research fresh
    rf = parse_research({"topic":"solar","summary":"...","sources":[],"generated_at":"2026-05-17T..."})
    assert rf["kind"] == "fresh"

    # Research cached
    rc = parse_research({"topic":"climate","summary":"...","sources":[],"generated_at":"2024-03-15","cached":True,"cache_age_seconds":99})
    assert rc["kind"] == "cached"

    # Research truncated
    rt = parse_research({"topic":"long...","summary":"...","sources":[],"generated_at":"...","truncated":True,"original_topic_length":62,"processed_topic":"first 50"})
    assert rt["kind"] == "truncated"

    # Research timeout
    re = parse_research({})
    assert re["kind"] == "timeout"

    print("All validations passed.")

if __name__ == "__main__":
    if "--validate" in sys.argv:
        validate()
    else:
        asyncio.run(main())

10What we're NOT building

  • skip Full test suite — --validate covers parsers; manual tests cover the rest
  • skip Fancy UI / Rich / curses — plain print() with flush=True
  • skip Voice / Deepgram / ElevenLabs (on-site scope)
  • skip Multi-provider router or LiteLLM — one SDK, one provider
  • skip Prompt-injection sanitiser (no active threat; JSON envelope is enough)
  • skip Background job queue or retry persistence
